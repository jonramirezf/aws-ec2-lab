\# AWS EC2 Lab – Conceptos Básicos



\## Introducción



En este laboratorio se estudiaron los conceptos fundamentales de AWS (Amazon Web Services) y su importancia en el mundo Cloud.



AWS es el proveedor de servicios en la nube líder a nivel mundial, ampliamente utilizado en entornos reales de producción.



\---



\## ¿Por qué aprender AWS?



\* Es el proveedor número 1 del mercado

\* Alta demanda laboral en Cloud y DevOps

\* Permite trabajar con infraestructura real



\---



\## Beneficios del Cloud Computing (AWS)



\### 1. Pagar solo por lo que usas



No es necesario invertir en hardware. Solo pagas por el consumo real de recursos.



\### 2. Escalabilidad bajo demanda



Puedes aumentar o disminuir recursos según la necesidad del sistema.



\### 3. Alta disponibilidad



Los servicios están diseñados para estar disponibles en múltiples regiones y zonas.



\### 4. Velocidad y agilidad



Puedes crear recursos en minutos (servidores, redes, almacenamiento).



\### 5. Seguridad



AWS ofrece herramientas avanzadas de seguridad y cumplimiento.



\### 6. Alcance global



Puedes desplegar aplicaciones en distintas partes del mundo fácilmente.



\---



\## Conceptos Clave



\### Escalabilidad



La escalabilidad es la capacidad de un sistema para adaptarse a un aumento de demanda.



\#### Tipos de escalabilidad



\*\*Vertical\*\*



\* Aumenta la potencia de una sola máquina

\* Más CPU, más RAM

\* Ejemplo: mejorar una instancia de EC2



\*\*Horizontal\*\*



\* Se agregan más servidores

\* Ejemplo: pasar de 1 servidor a 5 servidores



\---



\### Elasticidad



La elasticidad es la capacidad de un sistema para aumentar o disminuir recursos automáticamente según la demanda.



\*\*Ejemplo:\*\*



\* Aumenta el tráfico → AWS crea más instancias

\* Disminuye el tráfico → AWS elimina instancias



\---



\### Latencia



La latencia es el tiempo que tarda una solicitud en viajar desde el usuario hasta el servidor y volver.



\---



\### Flexibilidad



La flexibilidad es la capacidad de elegir y cambiar recursos fácilmente.



\*\*Ejemplo:\*\*



\* Hoy usas un servidor pequeño

\* Mañana cambias a uno más potente

\* Luego puedes migrar a otro tipo de servicio



\---
